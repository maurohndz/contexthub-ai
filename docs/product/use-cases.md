# Primary Use Cases

Ágora's use cases share one shape: *someone needs organizational context
they do not personally hold*. The difference is what they do with it.

## 1. Understand processes

> "How does the billing process currently work?"

Process manuals, operating procedures, and technical documentation are
loaded once; anyone authorized can then query how things actually work —
without interrupting the person who wrote them.

*Related chat mode:* **Explain process**.

## 2. Design requirements

> "Help me design this requirement considering existing business rules."

New requirements are drafted against the business rules already loaded in
the project, reducing contradictions discovered late.

*Related chat modes:* **Design requirement**, **Acceptance criteria**,
**Detect contradictions**.

## 3. Query meetings

> "What was decided in the last meeting about the payments module?"

Meeting minutes become queryable memory. A decision made verbally and
minuted is no longer trapped in the memory of whoever attended.

## 4. Impact analysis

> "How would this change affect the bank reconciliation process?"

Before changing a process or a system, query the loaded context for what
the change would touch — rules, processes, and prior decisions.

## 5. Onboarding

A new member joins a project and asks Ágora what the project does, how it
is organized, and which rules to respect — learning by asking instead of
by reading everything or booking meetings with busy people.

## 6. Knowledge preservation

When a member leaves, their documented contributions (manuals, minutes,
decision records) remain loaded and queryable. Their departure stops being
a knowledge event.

---

## Conversation modes available today

The chat offers task-specific modes that shape how the assistant uses the
retrieved context:

| Mode | Intent |
|---|---|
| General query | Open questions over the project's knowledge |
| Explain process | Walk through how a process works |
| Design requirement | Draft a requirement against existing rules |
| Summary | Summarize loaded material |
| Explain rules | Surface and explain applicable business rules |
| Detect contradictions | Find conflicts between sources or with a proposal |
| Acceptance criteria | Draft acceptance criteria for a requirement |
